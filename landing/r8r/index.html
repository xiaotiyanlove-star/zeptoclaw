Define Workflows

YAML, Code, or Visual. You Choose.

Define workflows declaratively with YAML, build visually with the drag-and-drop editor, or write custom nodes in Rust. All approaches are first-class citizens with 20+ built-in node types including Email, Slack, Database, and S3.

Explore the DSL
workflow.yaml
name: "Daily Report" trigger: schedule: "0 9 * * *" nodes: - name: "fetch_sales" type: "postgres/query" config: sql: | SELECT * FROM sales WHERE date > NOW() - INTERVAL '24h' - name: "format_report" type: "template" input: "{{ fetch_sales.rows }}" - name: "send_slack" type: "slack/post" config: channel: "#daily-reports"
